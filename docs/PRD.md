# 食譜收藏夾系統 PRD (Product Requirements Document)

## 1. 專案概述

### 背景與動機
現代人常在網路上看到各式各樣吸引人的食譜，但缺乏一個集中且有系統的地方來收藏與管理這些資訊。書籤整理不易，截圖又難以搜尋。「食譜收藏夾系統」旨在提供一個簡潔易用的平台，讓使用者可以輕鬆收藏、整理並隨時查閱自己喜愛的食譜，成為個人的線上數位食譜本。

### 目標用戶
- **一般人**：喜歡烹飪、有收藏及整理食譜需求的使用者。
- **管理員**：負責維護系統內容、管理使用者及確保平台運作正常之人員。

### 核心價值主張
提供直覺且方便的食譜收藏與檢索體驗，讓使用者能快速找到所需的食材清單與料理步驟，享受烹飪的樂趣。

---

## 2. 功能需求

### 主要功能清單與使用者故事

1. **儲存食譜**
   - 作為 **一般人**，我希望 **能夠新增或儲存我喜歡的食譜**，以便 **日後想要料理時可以隨時查看**。

2. **瀏覽食譜**
   - 作為 **一般人**，我希望 **能夠瀏覽我所有收藏的食譜清單**，以便 **在不知道要煮什麼時尋找靈感**。

3. **搜尋食譜**
   - 作為 **一般人**，我希望 **能夠透過關鍵字搜尋食譜**，以便 **在眾多收藏中快速找到特定的料理**。

4. **所需食材介紹**
   - 作為 **一般人**，我希望 **食譜內能詳細列出所需的食材及其份量**，以便 **在前往超市採買前確認需要準備哪些材料**。

5. **作法介紹**
   - 作為 **一般人**，我希望 **食譜內能呈現清晰的作法介紹與步驟**，以便 **在烹飪時能跟著指示一步步順利完成料理**。

6. **平台管理 (管理功能)**
   - 作為 **管理員**，我希望 **能夠查看、編輯或刪除平台上的所有食譜與管理使用者帳號**，以便 **維護系統的品質與內容適當性**。

---

## 3. 非功能需求

### 技術限制
- **前端技術**：HTML, CSS, JavaScript (必要時可搭配框架)。
- **後端與資料庫**：建議採用輕量級框架實作（如範例所提之 Flask + Jinja2 + SQLite），以加速開發與部署。

### 效能與安全考量
- **響應式設計 (RWD)**：介面需支援手機與電腦版瀏覽，方便使用者在廚房使用手機查看食譜。
- **安全性**：系統需具備基本的防護機制，如防止 SQL Injection 及 XSS 攻擊，保障使用者資料安全。
- **效能**：確保頁面載入速度流暢，尤其是食譜圖片（若有）的載入需進行最佳化。

---

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**
  - 儲存食譜（包含名稱、食材、作法）。
  - 瀏覽食譜清單。
  - 查看單一食譜的詳細內容（食材與作法介紹）。

- **Should Have (應該有)**
  - 搜尋食譜功能（依據食譜名稱或食材搜尋）。
  - 使用者能編輯或刪除自己儲存的食譜。

- **Nice to Have (加分項)**
  - 食譜分類、標籤功能。
  - 食譜圖片上傳與展示。
  - 管理員後台介面。
