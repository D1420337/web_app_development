# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述

### 背景與動機
現代人常在網路上看到喜歡的食譜，但缺乏一個統整、分類且易於管理的平台。為了讓使用者能輕鬆收藏、尋找並瀏覽自己喜歡的食譜，我們決定開發「食譜收藏夾系統」。

### 目標用戶
- **一般用戶**：想要收藏、瀏覽與搜尋日常食譜的美食愛好者或自煮族。
- **管理員**：負責維護系統運作、管理食譜內容與使用者帳號的管理者。

### 核心價值主張
提供一個直覺、簡單且專注於食譜收藏的平台，讓使用者不再遺失任何一道好菜的作法與食材比例，隨時隨地輕鬆下廚。

## 2. 功能需求

### 1. 儲存食譜
- **作為** 一般用戶，**我希望** 能夠新增並儲存新的食譜到我的收藏夾，**以便** 日後需要時能快速找到。
- **作為** 管理員，**我希望** 能夠審核或編輯平台上的公開食譜，**以便** 確保內容的正確性與品質。

### 2. 瀏覽食譜
- **作為** 一般用戶，**我希望** 能夠在首頁或收藏夾中以列表或卡片的方式瀏覽食譜，**以便** 尋找今天的料理靈感。

### 3. 搜尋食譜
- **作為** 一般用戶，**我希望** 能夠透過關鍵字（如菜名、食材）來搜尋食譜，**以便** 快速找到我想做的特定料理。

### 4. 所需食材介紹
- **作為** 一般用戶，**我希望** 在查看食譜時，能清楚看到完整的所需食材清單與份量，**以便** 我在料理前能夠準確準備食材。

### 5. 作法介紹
- **作為** 一般用戶，**我希望** 在查看食譜時，能看到詳細且按步驟條列的作法介紹，**以便** 我能跟著指示順利完成料理。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架。
- **前端模板**：使用 Jinja2 模板引擎進行頁面渲染，搭配基礎 HTML/CSS/JavaScript。
- **資料庫**：使用 SQLite 作為關聯式資料庫。

### 效能與安全考量
- **效能**：系統首頁及搜尋功能載入時間應在 3 秒內完成；頁面設計需具備基礎響應式（RWD）支援。
- **安全**：
  - 密碼需進行雜湊（Hash）處理後再儲存。
  - 實作防範 SQL Injection 與 XSS（跨站指令碼）攻擊的基本防護。
  - 確保管理員權限與一般用戶權限的存取控制隔離。

## 4. MVP 範圍

### Must Have (必須擁有 - 核心功能)
- 使用者註冊與登入（包含區別一般用戶與管理員）。
- 食譜的 CRUD 功能（建立、讀取、更新、刪除）。
- 食譜的詳細頁面展示（包含食材清單與步驟作法）。
- 基礎的食譜瀏覽與列表呈現。

### Should Have (應該擁有 - 重要附加功能)
- 食譜的關鍵字搜尋功能。
- 食譜分類標籤（如：中式、西式、甜點等）。

### Nice to Have (最好擁有 - 未來擴充功能)
- 食譜圖片上傳與展示功能。
- 使用者可對食譜留下評論或評分。
- 食譜匯出（如 PDF 格式）或分享至社群平台。
